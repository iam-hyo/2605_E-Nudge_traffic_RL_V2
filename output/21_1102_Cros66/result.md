# 0521 6x6 실험결과 3 — 엣지-상대적 행동공간 + 신호 우호 우회도로 채택 검증

> 실험일 2026-05-21 · 토폴로지 `6x6_cross_topology.json` · 경로 1→36 · peak 08:00 · 30회 반복

---

## 1. 서론

기존 RL 최소연료 경로탐색 실험(0520 6x6 실험결과 2)은 두 가지 한계가 있었다.

1. **행동 공간이 토폴로지에 종속** — 모델 출력층이 전역 노드 수(N)에 1:1로
   묶여 있어, 토폴로지가 바뀌면 모델을 처음부터 다시 학습해야 했고 여러
   도로망을 하나의 모델로 다룰 수 없었다 (사회적 실효성 = 일반화 불가).
2. **무신호 도로 악용** — 토폴로지에 신호 없는 도로가 있어 모델이 신호를
   학습하지 않고 무신호 우회로만 타는 행동으로 수렴했다.

본 실험은 두 가지를 동시에 개선한다.

- **엣지-상대적 행동 공간(edge-relative action space)** 도입 — 행동을
  "전역 노드 ID" 가 아니라 "현재 노드에서 방위순 k 번째 후보 엣지(슬롯)" 로
  재정의. 출력 차원이 토폴로지와 무관한 고정값(K_HOP1=4)이 되어, **하나의
  모델을 여러 토폴로지에서 동시에 학습**할 수 있게 되었다.
- **6x6_cross 토폴로지 재설계** — 36개 노드 전부에 신호를 부여(좌회전 phase
  포함)하여 무신호 악용을 차단하고, "가장 길지만 신호가 가장 우호적인 간선
  도로"(col 2, n3→n33)를 의도적으로 배치했다.

**핵심 질문**: 신호를 학습한 모델은, 거리가 가장 길더라도 신호 연동(green-wave)
간선도로를 선택하는가?

## 2. 토폴로지 분석

### 2.1 구조

6×6 = 36 노드. 6개의 세로 사선 도로(col) × 6개의 가로선(row). ID = row·6 + col + 1.
출발 n1(0,0) → 목적 n36(7,5). (좌표 단위 × 350 m)

| col | 진입(row0) → 종점(row5) | 성격 | 도로등급 |
|---|---|---|---|
| 0 | n1(0,0) → n31(0,5) | 신호 불리 (hostile) | local (저속) |
| 1 | n2(1,0) → n32(2.9,5) | **명시적 최단**·신호 불리, n2 좌회전 대기 김 | local (저속) |
| 2 | n3(4,0) → n33(3.1,5) | **명시적 최장**·신호 우호 green-wave (우회 간선) | arterial (고속) |
| 3 | n4(5,0) → n34(5.5,5) | 신호 약간 불친절 | medium |
| 4 | n5(6.5,0) → n35(6,5) | 신호 꽤 친절 (불완전) | medium |
| 5 | n6(7,0) → n36(7,5) | 신호 보통 | medium |

- **36/36 노드 전부 신호 보유** (각 노드 green·left_turn·red 3-phase). 무신호 악용 차단.
- col 1·2 의 정상부는 사용자 명세상 (3,5) 로 동일 → 길이 0 링크를 피하려고
  2.9 / 3.1 로 0.2 만큼만 분리 (사선이 정상부에서 거의 만나는 cross 형태 유지).

### 2.2 신호·속도 설계

| col | cycle | green 비율 | 위상 | 도로속도(peak) |
|---|---|---|---|---|
| 0 hostile | 180s | 13% | 매 노드 red 도착 (anti-wave) | ~8.5 km/h |
| 1 hostile | 220s | 7% | 매 노드 red 도착, 진입 좌회전 난항 | ~8.5 km/h |
| **2 friendly** | **70s** | **63%** | **green-wave 위상고정 → 무정차** | **~26 km/h** |
| 3 slight | 150s | 23% | red 중간 도착 | ~14 km/h |
| 4 quite | 110s | 38% | red 후반 도착 (짧은 정지) | ~14 km/h |
| 5 moderate | 170s | 16% | red 후반 도착 | ~14 km/h |

현실 서사: **col 2 는 신호 연동된 빠른 간선도로**, col 0·1 은 정체된 느린
이면도로다. col 2 는 거리는 가장 길지만 (1) green-wave 무정차 (2) 높은 주행
속도라는 두 이점을 갖는다.

### 2.3 사전 검증 (강제 주행, peak 60회 평균)

각 col 도로로 1→36 을 강제 주행시킨 결과:

| col | 성격 | 거리(m) | 연료(mL) | 대기(s) |
|---|---|---|---|---|
| 0 | hostile | 4200 | 1016 | 651 |
| 1 | hostile | **3657 (최단)** | 1023 | 862 |
| **2** | **friendly** | **4543 (최장)** | **824 (최소)** ✅ | **363** |
| 3 | slight | 4033 | 934 | 691 |
| 4 | quite | 4383 | 912 | 526 |
| 5 | moderate | 4200 | 982 | 743 |

→ **거리가 가장 긴 col 2 가 연료는 가장 적다.** 최단거리(col 1) 대비 약 200 mL
(20%) 절감. 설계 의도대로 "거리 ↔ 신호 유불리" 가 명확히 분리됨.

## 3. 가설

- **H1** — 최단거리 모델(① shortest_dijkstra)은 거리가 가장 짧은 col 1 을
  선택하고, 신호 불리로 인해 연료가 가장 많다.
- **H2** — 연료·신호 인식 모델(②③④⑤)은 거리가 가장 긴 col 2 를 선택한다.
  green-wave 무정차 + 간선 고속이 거리 손해를 압도하기 때문이다.
- **H3** — 신호 정보를 더 잘 쓰는 모델일수록 연료가 적다 (⑤ ≥ ④ ≥ ③).

## 6. 결과 해석

### 6.1 가설 검증

- **H1 ✅** — shortest_dijkstra 는 예상대로 최단거리 col 1(3657 m)을 선택.
  신호 불리로 대기 909 s, 연료 1037 mL 로 **5개 모델 중 최악**.
- **H2 ✅** — RL 3종 + static_fuel 모두 **가장 긴 도로인 col 2 를 채택**.
  거리는 +24~27% 길지만 green-wave 무정차 + 간선 고속으로 대기를 909→360 s
  로 약 60% 줄여 연료를 21% 절감.
- **H3 ✅** — rl_signal_attention(⑤, 819 mL) < rl_signal(④, 836) <
  rl_base(③, 849). 신호 정보를 attention 으로 더 잘 활용한 모델이 가장 우수.
  ⑤ 는 연료뿐 아니라 소요 시간(1305 s)도 가장 짧다.

### 6.2 핵심 메시지

> **"가장 긴 도로인데도, 신호를 학습한 모델은 이 길을 선택했습니다."**
> rl_signal_attention 은 최단거리 모델 대비 **연료 21.0% 절감**
> (1037 → 819 mL), 동시에 거리는 900 m 더 멀고 소요 시간은 630 s 더 짧다.
> 단순 최단거리 탐색이 신호 정체를 무시해 오히려 비효율적임을 정량적으로 보였다.

### 6.3 엣지-상대적 행동 공간의 의의

기존 구조(출력 = 전역 노드 N개)는 토폴로지가 바뀔 때마다 재학습이 강제됐다.
본 실험의 엣지-상대적 행동 공간(출력 = 슬롯 4개 고정)은 이 종속을 끊어,
**6x6_cross · 10x10 · 강남구 3개 토폴로지를 하나의 모델로 동시 학습**할 수
있게 했다. 이는 "특정 지도에 특화된 모델" 에서 "여러 도로망에 적용 가능한
모델" 로 가는 기반이며, 현실 내비게이션으로의 확장 가능성(사회적 실효성)을
뒷받침한다.

### 6.4 한계 및 향후 과제

- 학습 시 강남구(1995 노드) 토폴로지는 무작위 경로의 도달률이 0%에 머물렀다.
  대규모 실도로망에서는 무작위 start/goal 의 도달 난이도가 높아, 보조 학습
  신호로는 기능했으나 본격적 일반화 평가에는 별도의 hold-out 토폴로지 평가가
  필요하다.
- col 4(quite-friendly)는 연료 912 mL 로 col 2(824)에 이어 2위였다. 신호
  우호도가 col 2 와 가까워, 노이즈가 큰 환경에서는 경합 가능성이 있다.
- 다음 단계: 미지(hold-out) 토폴로지에서의 성능 평가, GNN 기반 상태 인코더,
  엣지 비용 예측 모듈과 고전 경로탐색의 결합 비교.
