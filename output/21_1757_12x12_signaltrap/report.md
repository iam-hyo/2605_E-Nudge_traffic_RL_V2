# 12x12 중간 규모 테스트베드 — 신호 회피 경로탐색 실험

> 실험일 2026-05-21 · 토폴로지 `12x12_topology.json` (144 노드) · 경로 n1→n144 ·
> peak 08:00 · 30회 반복 · 5개 모델
> 산출물: `results.csv` · `summary.json` · `report.md` · `12x12_short_fdj_rlattn.gif`

---

## 1. 서론

6x6_cross(36 노드) 테스트베드에서 "신호를 학습한 모델이 거리는 길어도 신호가
우호적인 경로를 택한다"는 가설을 검증했다. 본 실험은 그 결과를 **강남구
실도로망(1995 노드)으로 전이하기 직전의 중간 단계**로, 규모와 현실 복잡도를
키운 **12x12 = 144 노드** 테스트베드에서 같은 효과가 성립하는지 확인한다.

**핵심 질문**: 규모가 4배로 커지고 신호·속도가 강남구 실측을 반영해도,
신호·연료 인식 모델은 "최단 거리" 함정을 피하고 우회 간선을 택하는가?

---

## 2. 환경 설계

### 2.1 토폴로지 (평가항목: 규모 · 현실 복잡도 · 사선도로)

| 항목 | 값 |
|---|---|
| 노드 | 144 (12×12 격자), ID = row·12 + col + 1 |
| 링크 | 255 (arterial 64 · medium 180 · local 11) |
| 출발 / 목적 | n1 (0,0) 좌하단 → n144 (11,11) 우상단 |
| 노드 간격 | 300 m 기본 + 위치 지터 ±20 m, 5개 열 사선화 |
| 최대 차수 | 4 (엣지-상대적 행동공간 K=4 슬롯과 정합) |

**구조의 핵심 — 대각 신호 함정 vs 간선 우회로**

- **대각 코리도** : n1→n14→n27→…→n144 의 사선 링크 11개. 기하학적 **최단
  (4672 m)** 이지만 — 모든 교차로가 3현시 대형 신호(TRA_LIGHT=4), 도로는
  local 저속 등급. "최단 = 신호지옥" 함정.
- **간선 우회로** : 행·열 {0, 6, 11} 은 arterial 등급(고속) + 무신호 다수.
  코너를 우회하면 거리는 +40 % 길지만 빠르고 신호 정체가 적다.
- 사선도로 — 주 대각 코리도(맵을 가로지르는 사선) + 5개 열 사선화로
  "격자 + 사선" 혼합. 현실 도시의 비격자성을 반영.

### 2.2 신호 현시 체계 (T-DATA SPAT 실측 기반 — 평가항목: 강남구 신호 반영)

명령문 환경1 로직을 그대로 구현. 노드별 TRA_LIGHT 유형 → 사이클·현시 샘플링.

| 유형 | 분포 | 사이클 (정규분포) | 현시 |
|---|---|---|---|
| `TRA_LIGHT=0` 무신호 | 47/144 | — | 신호 없음 |
| `TRA_LIGHT=3` 2현시 | 48/144 | N(μ=131, σ=30) clip[60,180] | green=cycle×0.321, red=나머지 |
| `TRA_LIGHT=4` 3현시 | 49/144 | N(μ=180, σ=30) clip[80,180] | green×0.256, left×0.144, red=나머지 |

- offset = `random.randint(0, cycle//2)` — 교차로 간 연동계획 데이터 부재 →
  사이클 절반 범위 균등 샘플링 (의도적 green-wave 없음 = 현실적 무작위 신호).
- 유형 배치: 대각 코리도 = 전부 3현시, 간선 교차로 = 3현시, 간선 통로 =
  무신호 위주, 내부 이면도로 = 중심부일수록 3현시 밀집.
- 본 실험은 위 분포에서 **1회 샘플링한 인스턴스를 고정**(seed 고정)해
  Dijkstra·재현성을 확보했다. 에피소드별 재샘플링은 향후 과제.

### 2.3 속도 생성 (평가항목: 강남구 속도 반영 · 현실성)

`gangnam_speed_data.csv` 경향(평균 ~28 km/h, 아침 첨두 정체)을 반영.

| 등급 | off-peak (07:00) | peak (08:05) | 비고 |
|---|---|---|---|
| arterial | 46 km/h | 25 km/h | 간선 — 첨두에 약 1.8배 감속 |
| medium | 35 km/h | 16 km/h | 보조간선 |
| local | 19 km/h | 9 km/h | 이면도로(코리도) |

- 24슬롯(5분, 07:00~08:55), slot 13(08:05) 첨두의 **비대칭 dip** — 완만한
  정체 형성 + 느린 회복(08:55 에도 미회복) = 강남구 아침 첨두 경향.
- 링크 길이 260~470 m, 속도 9~52 km/h — 한국 도심 간선/이면도로 현실값.

## 3. 사전 검증 (`util/verify_12x12.py`, 강제주행 40회 평균)

| 시간대 | 경로 | 거리(m) | 연료(mL) | 대기(s) |
|---|---|---|---|---|
| peak 08:00 | 대각 코리도(최단) | 4672 | 1233 | 434 |
| peak 08:00 | 간선 우회로 | 6608 | **918 (−25.5%)** | 201 |
| off_peak 07:00 | 대각 코리도(최단) | 4672 | 840 | 434 |
| off_peak 07:00 | 간선 우회로 | 6608 | 1023 | 383 |

→ **peak 에서는 우회로가 연료 25 % 우위** — 신호 함정이 성립.
→ off_peak 에서는 정체가 약해 최단 코리도가 유리 — **신호 회피의 이득은
   혼잡(첨두) 의존적**. 본 실험은 효과가 뚜렷한 peak 로 한정.

### 5.2 채택 경로

- **① shortest** : `1→14→27→40→53→66→79→92→105→118→131→144` — 대각 코리도를
  그대로 직진. 거리는 최단(4672 m)이나 11개 교차로 전부 3현시 대형신호 +
  local 저속 → 대기 451 s, 연료 1232 mL 로 **5개 모델 중 최악**.
- **②③④⑤** : 모두 `1→13→25→…→73`(좌측 col 0 간선 상행) → `73→74→…→77`
  (row 6 간선) → 격자 상행 → row 11 간선 으로 **대각 코리도를 회피**.
  거리는 6500~6800 m (+39~46 %) 길지만 연료는 12~20 % 적다.

## 6. 핵심 발견

1. **신호 회피 효과가 12x12 규모에서도 성립.** 최단거리 모델이 택한 대각
   코리도는 거리가 가장 짧지만(4672 m) 연료는 가장 많다(1232 mL). 신호·연료를
   인식한 4개 모델은 모두 +40 % 거리를 감수하고 우회 간선을 택해 연료를
   12~20 % 절감했다 — **거리 최소화는 신호 정체 환경에서 잘못된 목적함수**임을
   144노드 규모에서 재확인.

2. **static_fuel_dijkstra 가 최저 연료(991 mL, −19.6%).** 시간의존 Dijkstra 가
   링크별 속도 기댓값 + 신호 대기 기댓값을 정확히 알고 계획하므로 가장
   효율적이다 (환경 모델을 가진 고전 기법의 상한선).

3. **RL 중에서는 attention 이 최고 (1028 mL, −16.6 %, 도달 100 %).**
   ⑤ attention < ④ signal(1068) < ③ base(1083) — 신호 정보를 더 잘 활용한
   모델일수록 연료가 적다. attention 은 표준편차도 69 로 가장 작아 가장
   일관적이며, 환경 모델 없이 경험만으로 static_fuel 의 상한선에 근접했다.

4. **시간 단축 동반.** 우회로 모델들은 연료뿐 아니라 소요 시간도 짧다
   (shortest 2300 s → attention 1439 s, static_fuel 1383 s). 대각 코리도의
   저속·신호대기가 거리 이득을 압도한다.

## 7. 결론

12x12 = 144 노드, 강남구 실측 기반 신호·속도를 갖춘 중간 규모 테스트베드에서
**신호·연료 인식 경로탐색이 단순 최단거리 대비 연료를 최대 20 % 절감**함을
정량적으로 입증했다. 6x6_cross(36노드)의 결론이 4배 규모·현실 복잡도 증가
환경에서도 견고하게 유지된다. RL 모델은 환경 모델 없이 학습만으로 시간의존
Dijkstra 의 상한선에 근접했으며, attention 모델이 가장 우수했다. 이는 강남구
실도로망 전이의 타당성을 뒷받침하는 중간 결과다.

## 8. 한계 및 문제점

1. **attention 모델의 규모 확장성** — base·signal 은 4000 에피소드에서 도달률
   98~99 %에 수렴했으나 attention 은 동일 조건에서 60~80 %로 불안정,
   6000 에피소드 + 탐색 연장 후에야 100 %에 도달했다. 더 작은 attention
   구조가 144노드 탐색에 1.5배의 학습을 요구 — **1995노드 강남구 전이 시
   학습 비용·안정성의 핵심 리스크**.

2. **신호 회피 이득의 혼잡 의존성** — off_peak(07:00)에서는 정체가 약해 최단
   코리도가 오히려 유리(검증 §3). 신호 회피의 가치는 첨두에 한정되며, 본
   실험은 peak 만 평가했다. 시간대 조건부 라우팅(시간 인식 정책) 평가가 필요.

3. **좌회전 제약과 고전 플래너** — 2현시(TRA_LIGHT=3) 교차로는 좌회전을
   금지(현실 반영)하는데, StaticFuelDijkstra 가 이를 모델링하지 않아 초기엔
   통행 불가 경로를 계산해 0 % 도달했다. 회전 제약 인지 로직을 추가해
   해결 — **고전 플래너는 회전 제약을 명시 모델링해야 하나 RL 은 환경
   동역학에서 암묵적으로 학습**한다는 점을 시사.

4. **신호 인스턴스 고정** — 명령문은 에피소드별 사이클 재샘플링을 제시하나,
   Dijkstra 계열의 재현성을 위해 분포에서 1회 샘플링한 인스턴스를 고정했다.
   다양한 신호 환경에 대한 일반화 평가는 별도 과제.

5. **rl_signal 도달률 93 %** — 30회 중 2회 미도달. 연료 평균(1068 mL)이
   미도달분으로 다소 부풀려졌을 수 있다.
